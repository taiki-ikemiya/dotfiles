# research-report スキル

テーマについて調査し、構造化されたレポートを作成するスキル。

---

## フロー

以下の4ステップでレポートを作成する。

### ステップ1: ヒアリング

ユーザーの入力から以下を確認する。

- **テーマ**: 何について調査するか
- **目的**: なぜ調査するか（意思決定、知識共有、技術選定 等）
- **対象読者**: 誰に向けたレポートか（エンジニア、マネージャー、外部顧客 等）
- **出力形式**: Markdown か PPTX か
- **レポートの型**: 社内向けか対外向けか

入力から推定可能な項目はスキップし、不明な項目のみ質問する。
全て推定可能な場合はヒアリング自体をスキップして調査に進む。

### ステップ2: 調査

目的に応じて以下の手段を組み合わせる。

- **Web検索**: WebSearch / WebFetch を活用し、最新情報を取得する
- **コードベース調査**: Grep / Glob / Read を活用し、プロジェクト内の実装を調査する
- **ドキュメント調査**: 既存のドキュメントやREADMEを参照する

調査中は以下を記録する。

- 情報源のURL・ファイルパス
- 取得日時
- 情報の信頼度（公式ドキュメント > ブログ > フォーラム）

### ステップ3: ドラフト生成

調査結果を構造化されたレポートにまとめる。

- レポートの型（社内向け/対外向け）に応じたテンプレートを使用する
- `references/report-structure.md` のガイドラインに従う
- 比較調査の場合は必ず比較表を含める
- 結論と推奨アクションを明記する

### ステップ4: 最終成果物

出力形式に応じて仕上げる。

- **Markdown出力**: そのまま最終成果物として出力する
- **PPTX出力**: pptx-builder スキルと連携して生成する

---

## 出力形式

| 形式 | 用途 | 方法 |
|---|---|---|
| Markdown | 社内共有・ドキュメント蓄積 | 直接生成 |
| PPTX | 報告会・対外共有 | pptx-builder スキルと連携 |

### Markdown出力

- 直接Markdownファイルとして出力する
- テンプレートに従った構造化されたフォーマットを使用する
- コード例、表、リンクを適宜含める

### PPTX出力

- まずMarkdownでドラフトを作成する
- 構造化データに変換し、pptx-builder スキルに引き渡す
- 引き渡しデータ形式は `references/output-formats.md` を参照する

---

## レポートの型

### 社内向けレポート

- 技術詳細を多めに記述する
- コード例やコマンド例を含める
- 体裁は最低限でよい（内容重視）
- 対象読者はエンジニアやテックリードを想定する
- テンプレート: `assets/templates/internal-report.md`

### 対外向けレポート

- エグゼクティブサマリーを重視する
- 図表を中心に据え、視覚的に分かりやすくする
- 専門用語は控えめにし、必要な場合は注釈を付ける
- 対象読者はマネージャーや顧客を想定する
- テンプレート: `assets/templates/external-report.md`

## 出力テンプレート（社内向け）

```markdown
# [タイトル]

## エグゼクティブサマリー
> 3行以内で結論を述べる

## 背景と目的
...

## 調査方法
- 調査範囲: ...
- 情報源: ...

## 調査結果
### [観点1]
...
### [観点2]
...

## 比較・分析
| 項目 | A | B | C |
|---|---|---|---|

## 結論と推奨
...

## Next Steps
- [ ] ...

## 参考文献
- ...
```

## 連携

### idea-forge との連携

- idea-forge で有望な仮説が得られた場合、research-report で本格的な調査を行う
- idea-forge の出力（仮説リスト、評価結果）を調査のインプットとして活用する

### pptx-builder との連携

- PPTX出力が必要な場合、pptx-builder スキルに構造化データを渡す
- 引き渡しデータ形式の詳細は `references/output-formats.md` を参照する
- Markdownドラフトをスライド構成に変換してから引き渡す

---

## 検索ソースの明記

WebSearch や WebFetch で検索・取得した情報は、必ず以下を明記する。

- 情報源のURL
- 取得日
- 情報の概要

参考文献セクションに以下の形式で記載する。

```
- [タイトル](URL) - 取得日: YYYY-MM-DD
```

## 品質チェックリスト

レポート完成時に以下を確認する。

- [ ] エグゼクティブサマリーが3行以内で結論を述べている
- [ ] 調査方法と情報源が明記されている
- [ ] 比較調査の場合、比較表が含まれている
- [ ] 結論と推奨アクションが明確である
- [ ] Next Steps が具体的なタスクとして記載されている
- [ ] 参考文献に全ての情報源が記載されている
- [ ] Web検索で取得した情報にはソースURLが付記されている
- [ ] 対象読者に適した詳細度・用語レベルになっている
