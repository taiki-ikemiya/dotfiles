# スライドレイアウト一覧

pptx-builder で利用可能なレイアウトの詳細リファレンス。

---

## 1. title_slide（タイトルスライド）

プレゼンテーションの表紙として使用する。

### 用途
- プレゼンテーションの先頭（表紙）

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | はい | メインタイトル（28pt、Bold、中央揃え） |
| `subtitle` | string | いいえ | サブタイトル（16pt、subtext色、中央揃え） |
| `author` | string | いいえ | 作成者名（10pt、下部中央） |
| `date` | string | いいえ | 日付（10pt、下部中央） |

### 配置

```
+------------------------------------------+
|                                          |
|                                          |
|         プレゼンタイトル                   |
|            ─────                         |
|           サブタイトル                     |
|                                          |
|                                          |
|          日付 | 作成者名                   |
+------------------------------------------+
```

### パターンごとの差異

| 属性 | soft_business | clean_pro | warm_formal |
|---|---|---|---|
| 背景 | bg | bg | warm |
| タイトル色 | primary | text | primary |

### 入力例

```json
{
  "layout": "title_slide",
  "title": "2026年度 事業計画",
  "subtitle": "経営企画部",
  "author": "山田太郎",
  "date": "2026-02-28"
}
```

### 備考
- トップレベルの `title` と `subtitle` がある場合、最初のスライドとして自動生成される
- `slides` 配列の先頭が `title_slide` の場合、自動生成はスキップされる
- タイトルとサブタイトルの間に `primary` の短い横線が描画される

## 2. toc（目次スライド）

全体構成を俯瞰するためのスライド。

### 用途
- プレゼン全体の目次・アジェンダ

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | いいえ | 見出し（デフォルト: "目次"） |
| `items` | array | はい | 目次項目の配列 |
| `items[].number` | string | いいえ | 番号（デフォルト: 自動連番 "01", "02"...） |
| `items[].title` | string | はい | セクション名 |
| `items[].page` | number/string | いいえ | ページ番号 |

### 配置

```
+------------------------------------------+
|  目次                                     |
|  ─────                                   |
|                                          |
|   01  セクションタイトル .............. 3  |
|   02  セクションタイトル .............. 8  |
|   03  セクションタイトル ............. 14  |
|   04  セクションタイトル ............. 20  |
|                                          |
+------------------------------------------+
```

### 入力例

```json
{
  "layout": "toc",
  "title": "目次",
  "items": [
    {"number": "01", "title": "プロジェクト概要", "page": 3},
    {"number": "02", "title": "現状分析", "page": 8},
    {"number": "03", "title": "提案内容", "page": 14},
    {"number": "04", "title": "スケジュール", "page": 20}
  ]
}
```

## 3. section_divider（セクション区切りスライド）

新しいセクションの冒頭に使用。

### 用途
- セクション間の区切り
- 新しいトピックへの遷移

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `number` | string | いいえ | セクション番号（40pt、primary、Bold） |
| `title` | string | はい | セクションタイトル（26pt、text、Bold） |
| `description` | string | いいえ | 概要テキスト（14pt、subtext） |

### 配置

```
+------------------------------------------+
|                                          |
|                                          |
|  ■                                       |
|  01                                      |
|  セクションタイトル                        |
|  ── セクションの概要テキスト ──            |
|                                          |
|                                          |
+------------------------------------------+
```

### パターンごとの差異

| 属性 | soft_business | clean_pro | warm_formal |
|---|---|---|---|
| 背景 | bg | bg + 左に primary 縦線 | warm 全面 |

### 入力例

```json
{
  "layout": "section_divider",
  "number": "01",
  "title": "プロジェクト概要",
  "description": "背景・目的・スコープの説明"
}
```

## 4. content_text（テキスト主体コンテンツ）

説明文・箇条書き・ポイント解説に使用。

### 用途
- 一般的な情報の提示
- 箇条書きによる要点の列挙
- 説明文の段落表示

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | はい | スライドタイトル |
| `text` | string | いいえ | 段落テキスト（14pt） |
| `content` | array[string] | いいえ | 箇条書き項目（14pt、`bullets` のエイリアス） |
| `bullets` | array[string] | いいえ | 箇条書き項目（`content` と同じ） |

`text` と `content`/`bullets` は併用可能（text が先、bullets が後に表示）。

### 配置

```
+------------------------------------------+
| [パターン別ヘッダー]                       |
| セクション見出し                           |
| ─────────────                            |
|                                          |
|  本文テキストが入ります。                   |
|                                          |
|  ● ポイント1の説明テキスト                 |
|  ● ポイント2の説明テキスト                 |
|  ● ポイント3の説明テキスト                 |
|                                          |
+------------------------------------------+
```

### 入力例

```json
{
  "layout": "content_text",
  "title": "プロジェクト概要",
  "content": [
    "目的: 業務効率化による生産性向上",
    "期間: 2026年4月〜2026年9月",
    "予算: 500万円",
    "担当: DX推進チーム（5名）"
  ]
}
```

### 備考
- 箇条書きには primary 色の小円マーカーが付与される
- 項目数は 4〜6 個を推奨する
- `title_content` レイアウトは `content_text` の後方互換エイリアス

## 5. content_cards（カード型コンテンツ）

サービス紹介・比較・特徴一覧に使用。2〜4カードの横並び。

### 用途
- サービス・機能の紹介
- 特徴の比較一覧
- ステップ紹介

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | はい | スライドタイトル |
| `cards` | array | はい | カード配列（2〜4枚） |
| `cards[].label` | string | はい | カードラベル（12pt、SemiBold） |
| `cards[].description` | string | いいえ | カード説明文（10pt、subtext） |

### 配置

```
+------------------------------------------+
| [パターン別ヘッダー]                       |
| セクション見出し                           |
|                                          |
|  ┌─────────┐ ┌─────────┐ ┌────────┐     |
|  │  ●       │ │  ●       │ │  ●      │     |
|  │ ラベル    │ │ ラベル    │ │ ラベル   │     |
|  │ 説明文   │ │ 説明文    │ │ 説明文   │     |
|  └─────────┘ └─────────┘ └────────┘     |
|                                          |
+------------------------------------------+
```

### パターンごとの差異

| 属性 | soft_business | clean_pro | warm_formal |
|---|---|---|---|
| カード背景 | white | white | warm |
| カード枠線 | border | border | なし |
| 角丸 | やや丸い | シャープ | やや丸い |
| アイコン形状 | 円形 | 角丸四角 | 円形 |

### 入力例

```json
{
  "layout": "content_cards",
  "title": "サービスの特徴",
  "cards": [
    {"label": "スピード", "description": "最短3日で導入可能"},
    {"label": "コスト", "description": "初期費用無料プランあり"},
    {"label": "サポート", "description": "24時間対応のサポート体制"}
  ]
}
```

### 備考
- カード数は 3 枚を推奨する（バランスが最も良い）
- アイコンは primary / secondary / accent を順に使い分ける
- カード間ギャップは自動調整される

## 6. content_data（KPI・数値ハイライト）

KPI・グラフ・数値ハイライトの表示に使用。

### 用途
- KPI・成果指標の表示
- 数値データのハイライト
- 実績報告

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | はい | スライドタイトル |
| `metrics` | array | はい | KPI指標の配列（最大4つ） |
| `metrics[].value` | string | はい | 数値（40pt、primary、Bold） |
| `metrics[].label` | string | はい | ラベル（10pt、subtext） |
| `description` | string | いいえ | 補足テキスト（数値下に表示） |

### 配置

```
+------------------------------------------+
| [パターン別ヘッダー]                       |
| セクション見出し                           |
|                                          |
|     ┌──────┐  ┌──────┐  ┌──────┐        |
|     │ 120% │  │ 3.2万│  │ 98%  │        |
|     │ 売上  │  │ ユーザ│  │ 満足度│        |
|     └──────┘  └──────┘  └──────┘        |
|                                          |
|  補足テキスト                              |
|                                          |
+------------------------------------------+
```

### 入力例

```json
{
  "layout": "content_data",
  "title": "主要KPI",
  "metrics": [
    {"value": "120%", "label": "売上達成率"},
    {"value": "3.2万", "label": "月間ユーザー数"},
    {"value": "98%", "label": "顧客満足度"}
  ],
  "description": "2026年1月時点の実績値"
}
```

## 7. content_table（テーブル・比較表）

プラン比較・機能比較・Before/After に使用。

### 用途
- プラン・機能の比較表
- Before / After の対比
- 一覧データの表示

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | はい | スライドタイトル |
| `headers` | array[string] | はい | テーブルヘッダー列 |
| `rows` | array[array] | はい | データ行の配列 |
| `highlight_cells` | array[array] | いいえ | 強調表示するセル座標 `[[行, 列], ...]` |

### 配置

```
+------------------------------------------+
| [パターン別ヘッダー]                       |
| 比較タイトル                              |
|                                          |
|  ┌──────────┬──────────┬──────────┐      |
|  │ ヘッダー  │ プランA   │ プランB  │      |
|  ├──────────┼──────────┼──────────┤      |
|  │ 機能1    │ ●        │ ●        │      |
|  │ 機能2    │ ●        │ –        │      |
|  │ 機能3    │ –        │ ●        │      |
|  └──────────┴──────────┴──────────┘      |
|                                          |
+------------------------------------------+
```

### パターンごとの差異

| 属性 | soft_business | clean_pro | warm_formal |
|---|---|---|---|
| ヘッダー背景 | warm | primary | primary |
| ヘッダー文字色 | text | white | white |
| セル背景 | 偶数行 bg / 奇数行 white | 同左 | 同左 |

### 入力例

```json
{
  "layout": "content_table",
  "title": "プラン比較",
  "headers": ["機能", "スタンダード", "プレミアム"],
  "rows": [
    ["基本機能", "●", "●"],
    ["高度な分析", "–", "●"],
    ["優先サポート", "–", "●"],
    ["API連携", "●", "●"]
  ],
  "highlight_cells": [[1, 2], [2, 2]]
}
```

## 8. closing（クロージングスライド）

プレゼン末尾。CTA・連絡先を提示。

### 用途
- プレゼンの締めくくり
- 連絡先・次のアクションの提示

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `message` | string | いいえ | メインメッセージ（デフォルト: "ありがとうございました"） |
| `title` | string | いいえ | `message` のエイリアス |
| `submessage` | string | いいえ | サブメッセージ（14pt、subtext） |
| `cta` | string | いいえ | CTAボタンテキスト |
| `contact` | array[string] | いいえ | 連絡先情報（複数行） |

### 配置

```
+------------------------------------------+
|                                          |
|                                          |
|     ありがとうございました                 |
|     ── ひとこと添えるメッセージ ──         |
|                                          |
|     ┌──────────────────────┐             |
|     │  まずはご相談ください  │             |
|     └──────────────────────┘             |
|                                          |
|     mail@example.com                     |
|     https://example.com                  |
|                                          |
+------------------------------------------+
```

### パターンごとの差異

| 属性 | soft_business | clean_pro | warm_formal |
|---|---|---|---|
| 背景 | bg | bg | warm |
| CTAボタン | primary角丸 | text角丸 | primaryピル型 |

### 入力例

```json
{
  "layout": "closing",
  "message": "ありがとうございました",
  "submessage": "ご質問がございましたらお気軽にお申し付けください",
  "cta": "まずはご相談ください",
  "contact": [
    "mail@example.com",
    "https://example.com"
  ]
}
```

## 9. two_column（2カラム比較）

画面を左右2カラムに分割し、比較や対比を表現する。

### フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `title` | string | はい | スライドタイトル |
| `left` | object | はい | 左カラムデータ |
| `left.header` | string | はい | 左カラムの見出し |
| `left.items` | array[string] | はい | 左カラムの箇条書き |
| `right` | object | はい | 右カラムデータ |
| `right.header` | string | はい | 右カラムの見出し |
| `right.items` | array[string] | はい | 右カラムの箇条書き |

### 入力例

```json
{
  "layout": "two_column",
  "title": "導入方式の比較",
  "left": {
    "header": "クラウド方式",
    "items": ["初期費用が低い", "スケーラビリティが高い", "運用負荷が小さい"]
  },
  "right": {
    "header": "オンプレミス方式",
    "items": ["カスタマイズ性が高い", "セキュリティを自社管理", "ランニングコストが低い"]
  }
}
```

|---|---|---|---|
| `title` | string | いいえ | スライドタイトル |
| `text` | string | いいえ | 自由テキスト |

### 入力例

```json
{
  "layout": "blank",
```

## デザイン仕様まとめ

### フォント

| 要素 | フォント | サイズ | 色 | スタイル |
|---|---|---|---|---|
| メインタイトル | Noto Sans JP | 28pt | パターン依存 | Bold |
| サブタイトル | Noto Sans JP | 16pt | subtext | Regular |
| セクション見出し | Noto Sans JP | 22pt | パターン依存 | Bold |
| 本文 | Noto Sans JP | 14pt | text | Regular |
| キャプション | Noto Sans JP | 10pt | subtext | Regular |
| KPI数値 | Noto Sans JP | 40pt | primary | Bold |
| カードラベル | Noto Sans JP | 12pt | text | SemiBold |
| カード説明 | Noto Sans JP | 10pt | subtext | Regular |

### カラーパレット

| トークン | HEX | 説明 |
|---|---|---|
| primary | #C07A5A | テラコッタブラウン |
| secondary | #8FBA78 | ナチュラルグリーン |
| accent | #6B8FA4 | スレートブルー |
| warm | #F2DED8 | ピンクベージュ |
| text | #3B3533 | ダークブラウン |
| bg | #F9F8F7 | ウォームグレー |
| subtext | #9A8E8B | 補足テキスト色 |
| border | #E4DFDC | 枠線・区切り線 |

### スライドサイズ

- **アスペクト比**: 16:9
- **幅**: 13.333 インチ
- **高さ**: 7.5 インチ
