# 出力テンプレート詳細定義

idea-forge の各深度レベルにおける出力フォーマットの完全版。

---

## quick 出力テンプレート

ターミナルに直接表示する。ファイル保存は行わない。
全体で30秒以内に読める分量に収めること。

### フォーマット

```
📌 論点: [論点A], [論点B], [論点C]
🔍 穴: [論点X]の前提が未検証、[論点Y]に反例あり
💡 仮説: H1「[仮説1]」(高), H2「[仮説2]」(中), H3「[仮説3]」(低)
👉 Next: [具体的なアクション1]、[具体的なアクション2]
```

### 記述ルール

- 論点は3〜5個。それ以上ある場合は重要度順に絞る
- 穴は致命的なもののみ。細かい指摘は省略してよい
- 仮説は2〜4個。確からしさを括弧内に付与する
- Next は「考える」「検討する」ではなく、具体的な行動を書く
  - 良い例: 「競合3社の料金体系を調べる」「チームリードに工数感を聞く」
  - 悪い例: 「もう少し考える」「検討を続ける」
- 全体を4行に収める。補足が必要な場合は最大2行追加してよい

### 拡張フォーマット（補足が必要な場合）

```
📌 論点: [論点A], [論点B], [論点C]
🔍 穴: [論点X]の前提が未検証、[論点Y]に反例あり
💡 仮説: H1「[仮説1]」(高), H2「[仮説2]」(中), H3「[仮説3]」(低)
   ↳ H1とH2は排他的。H3はH1の部分修正案
👉 Next: [具体的なアクション1]、[具体的なアクション2]
   ↳ まずH1の検証から着手（コスト低）
```

## deep 出力テンプレート

mdファイルとして保存する。
ファイル名: `idea-forge-YYYYMMDD-[slug].md`
保存先: カレントディレクトリ

### フォーマット

```markdown
# 💡 [入力内容から自動生成したタイトル]

> 生成日: YYYY-MM-DD
> 深度: deep
> 入力文字数: [n]文字

---

## 元の考え

> [入力をそのまま引用。改行・書式もそのまま保持する]

---

## 論点の分解

入力から以下の論点を抽出した。

### 論点1: [論点タイトル]

- **種別**: 主張 / 前提 / 観察 / 疑問
- **内容**: [論点の説明]
- **依存関係**: [他の論点との関係。なければ「独立」]

### 論点2: [論点タイトル]

- **種別**: 主張 / 前提 / 観察 / 疑問
- **内容**: [論点の説明]
- **依存関係**: [他の論点との関係。なければ「独立」]

### 論点3: [論点タイトル]

（必要に応じて追加）

### 暗黙の前提

入力には直接書かれていないが、存在する前提:

- [前提1]
- [前提2]

## 検証結果

### 論点1: [論点タイトル]

- ✅ **妥当な点**: [根拠とともに記述]
- ⚠️ **要注意**: [懸念点、未検証事項]
- 🔗 **情報源**: [web検索した場合のソース。しなかった場合は省略]

### 論点2: [論点タイトル]

- ✅ **妥当な点**: [根拠とともに記述]
- ⚠️ **要注意**: [懸念点、未検証事項]

（論点ごとに繰り返す）

> research-report で本格調査しますか？
```

### 記述ルール

- 各セクションは空行で区切る
- 検証結果では事実と推測を明確に区別する
- 仮説マップは最低2行、最大5行
- Next Actions は最低2つ、最大5つ。すべて具体的な行動にする
- web検索を行った場合はソースURLを明記する

### share 固有の記述ルール

- 背景・文脈セクションは必須。第三者が前提知識なしで理解できる水準にせよ
- 仮説マップに「チームへの問いかけ」列を必ず追加せよ
- Next Actions には担当候補を括弧書きで添えよ。不明な場合は「要相談」とせよ
- 議論のためのガイドセクションを追加し、チームでの活用方法を示せ
- 文体はですます調を基本とする（チーム共有のため）
- 略語や社内用語を使う場合は初出時に説明を添えよ
